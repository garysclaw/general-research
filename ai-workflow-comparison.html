AI Workflow: Nanoclaw vs Remote Claude vs Local Model

Which AI assistant should you reach for — and when? A practical breakdown for your RTX 2070 + Telegram + Claude stack.

Updated April 2026 RTX 2070 · 8GB VRAM Windows + WSL2 Claude Sonnet 4.6 / Ollama

⚡ The Three Approaches

📡

Nanoclaw

Always-On Messaging Agent

Your custom Claude agent accessible via Telegram and other messaging platforms. Runs on Anthropic's API, maintains persistent conversation memory, can schedule tasks and push to GitHub — all while you're away from the keyboard.

Always-On Persistent Memory GitHub Integration Scheduled Tasks API Cost No File System
☁️

Remote Claude

Claude.ai / Claude Code / API

Anthropic's flagship models accessed directly — via the web UI, the Claude Code CLI (now increasingly restricted), or raw API. Best raw intelligence, strongest coding ability, but session-based and tightening restrictions on agentic use.

Highest Quality File System Access Claude Sonnet/Opus Session-Based Subscription or API Restrictions Tightening
🖥️

Local Model

Ollama + RTX 2070

Run Llama, Qwen, or Mistral locally via Ollama. Your RTX 2070 with 8GB VRAM handles 7–9B parameter models at Q4_K_M quantization comfortably — ~40+ tok/s. Zero ongoing cost, 100% private, works offline.

100% Private $0 Ongoing Offline VS Code / Continue.dev Lower Quality Setup Required

🗺️ Use Case Guide — When to Use Which

"I want to research something while I'm away from my desk"
📡 → Nanoclaw
Message it from Telegram, let it search and summarize, push results to GitHub. You don't need to be at your machine.
"I'm actively coding and need file edits and terminal access"
☁️ → Remote Claude / Claude Code
Best intelligence for complex refactors. Claude Code can read/write files and run bash — just watch for the tightening restrictions.
"I'm working with sensitive or private data"
🖥️ → Local Model
Nothing leaves your machine. Ollama + Qwen3/Llama for private documents, internal code, or anything you can't send to Anthropic.
"I want to ask questions all day without paying per token"
🖥️ → Local Model
Your RTX 2070 at ~$0.10/day electricity handles unlimited queries. Great for repetitive tasks, code completion, drafting.
"I need the smartest possible answer to a hard problem"
☁️ → Remote Claude (Opus)
Claude Opus 4.7 is still the strongest reasoning model available. Use it for architecture decisions, complex debugging, writing.
"I want scheduled tasks and push notifications"
📡 → Nanoclaw
Only option with native scheduling and Telegram delivery. Set it and forget it — get a message when the work is done.
"I want AI code completion while I type in VS Code"
🖥️ → Local Model + Continue.dev
Install Continue.dev extension, point it at Ollama on localhost:11434. Free, fast, private autocomplete with Qwen2.5-Coder.
"I want to document and push research while I sleep"
📡 → Nanoclaw
Schedule tasks overnight. By morning, the HTML page is built and pushed to GitHub. You wake up to a Telegram notification.
"Claude Code keeps getting blocked / restricted"
🖥️ + 📡 → Local + Nanoclaw
Local model handles file-level work via Continue.dev. Nanoclaw handles research and orchestration. Less dependency on Claude Code CLI.

Step 1 — Install Ollama on Windows (outside WSL2)

  1. Download OllamaSetup.exe from https://ollama.com/download/windows
  2. Run installer as Administrator — Ollama starts automatically in system tray
  3. Open PowerShell or Command Prompt and verify: ollama --version
  4. Pull your first model: ollama pull qwen3:8b
  5. Test it: ollama run qwen3:8b "Hello, are you running on GPU?"
  6. Check GPU usage in Task Manager → Performance → GPU while running
# Quick start commands
ollama pull qwen3:8b
ollama pull qwen2.5-coder:7b
ollama pull llama3.1:8b

# List installed models
ollama list

# Run interactive chat
ollama run qwen3:8b

# Check API is running
curl http://localhost:11434/api/tags

Step 2 — Install Open WebUI (browser interface)

  1. Install Docker Desktop for Windows if not already installed
  2. Run the Open WebUI container (auto-connects to Ollama on host):
docker run -d -p 3000:8080 \
  --add-host=host.docker.internal:host-gateway \
  -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \
  -v open-webui:/app/backend/data \
  --name open-webui \
  ghcr.io/open-webui/open-webui:main
  1. Open browser at http://localhost:3000
  2. Register a local account (first user = admin)
  3. Select your Ollama model from the dropdown and start chatting

Step 3 — Connect to VS Code via Continue.dev

  1. In VS Code, open Extensions (Ctrl+Shift+X) and search Continue
  2. Install Continue - Codestral, Claude, and more by Continue.dev
  3. Open Command Palette (Ctrl+Shift+P) → Continue: Open Config File
  4. Edit .continue/config.json to add Ollama as provider:
{
  "models": [
    {
      "title": "Qwen2.5-Coder (Local)",
      "provider": "ollama",
      "model": "qwen2.5-coder:7b",
      "apiBase": "http://localhost:11434"
    },
    {
      "title": "Qwen3 8B (Local)",
      "provider": "ollama",
      "model": "qwen3:8b",
      "apiBase": "http://localhost:11434"
    }
  ],
  "tabAutocompleteModel": {
    "title": "Qwen2.5-Coder Autocomplete",
    "provider": "ollama",
    "model": "qwen2.5-coder:7b"
  }
}
  1. Press Cmd+L / Ctrl+L to open chat, or just start typing — tab autocomplete activates inline
  2. Use @codebase to let Continue index your project for context-aware suggestions

Note for WSL2 Users — Reaching Windows Ollama from Linux

  1. Ollama runs on Windows (outside WSL2) and listens on port 11434
  2. From WSL2, reach it via the host IP: http://$(cat /etc/resolv.conf | grep nameserver | awk '{print $2}'):11434
  3. Or set: export OLLAMA_HOST=http://host.docker.internal:11434 in WSL2
  4. You can also set OLLAMA_ORIGINS=* in Windows environment variables to allow cross-origin access

⚠️ The Claude Code Situation

Why Claude Code / OpenClaw Is Getting Restricted

Anthropic has been progressively tightening access to Claude via third-party tools throughout 2025–2026. The root cause: community-led autonomous coding loops ("Ralph Wiggum" phenomenon in late December 2025) generated enormous token consumption, forcing Anthropic to crack down on unlimited agentic loops and subscription abuse.

Jan 9, 2026
Anthropic flipped a switch — third-party tools using Claude Pro/Max subscriptions stopped working overnight. Technical safeguards prevent spoofing of official Claude Code client.
Feb 2026
Official compliance docs ban consumer OAuth tokens (Free/Pro/Max) in any third-party tool — including Anthropic's own Agent SDK. OpenClaw explicitly named.
Apr 4, 2026
Claude Pro and Max subscribers can no longer use their subscription limits to power tools like OpenClaw. Cost increase of up to 50x for affected users.

What this means for David: If you've been using OpenClaw/Claude Code on a Pro subscription, that path is now blocked. Direct API access (what Nanoclaw uses) still works but is billed per token. The safest long-term stack is: Nanoclaw (API) + Local Ollama for daily use, with Remote Claude API reserved for complex tasks.

Official Claude Code CLI (the one Anthropic ships) still functions but is subject to evolving agentic loop restrictions. The Claude Code auto mode (released early 2026) attempts to balance capability with safety guardrails.
